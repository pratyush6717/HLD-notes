One line: Instead of returning null → return a safe object that does nothing (or provides default behaviour).

Root insight: Returning null = design problem. Null Object = design solution. 
// BAD — null check everywhere
Vehicle v = factory.getVehicle("truck");
if (v != null) {
  v.start(); // easy to miss → NPE
}
// Repeated in 50 places across codebase
// GOOD — Null Object Pattern
Vehicle v = factory.getVehicle("truck");
v.start(); // always safe — no null check needed
// NullVehicle.start() just does nothing
Problems with null returns
✗ Too many null checks everywhere
✗ Ugly, cluttered code
✗ Easy to miss one → NPE
✗ Hard to test and maintain
With Null Object
✔ Zero null checks
✔ Clean polymorphic code
✔ NPE impossible
✔ Easy to extend
Important: Null Object ≠ always do nothing. It can: do nothing silently, return default values, log a warning, or provide fallback behaviour. It's a safe implementation, not necessarily an empty one.
// Interface — same contract for all
interface Vehicle {
  void start();
  void stop();
  String getType();
}

// Real objects
class Car implements Vehicle {
  public void start()   { System.out.println("Car starting..."); }
  public void stop()    { System.out.println("Car stopping..."); }
  public String getType() { return "Car"; }
}
class Bike implements Vehicle {
  public void start()   { System.out.println("Bike starting..."); }
  public void stop()    { System.out.println("Bike stopping..."); }
  public String getType() { return "Bike"; }
}
// Null Object — safe dummy
class NullVehicle implements Vehicle {
  public void start()  { /* do nothing — or log warning */ }
  public void stop()   { /* do nothing */ }
  public String getType() { return "No vehicle"; }
}

// Factory — returns NullVehicle instead of null
class VehicleFactory {
  static Vehicle getVehicle(String type) {
    if (type.equals("car"))  return new Car();
    if (type.equals("bike")) return new Bike();
    return new NullVehicle(); // 🔥 never return null
  }
}
// Client — clean, no null checks ever
Vehicle v = VehicleFactory.getVehicle("truck");
v.start();    // NullVehicle.start() — safe, no crash
v.stop();     // NullVehicle.stop() — safe, no crash
v.getType();  // "No vehicle" — meaningful default
Null Object shows up everywhere in real systems — often without people realising it's a named pattern.
Logging — NullLogger
In tests or environments where logging is disabled — instead of checking if logger is null, inject a NullLogger.
class NullLogger implements Logger {
  log(msg) {} // silently ignore
}
Auth — GuestUser
When user is not logged in — return GuestUser instead of null. All user.hasPermission() calls still work safely.
class GuestUser implements User {
  boolean hasPermission(p) { return false; }
  String getName() { return "Guest"; }
}
Payment — NullProcessor
Free tier users — NullPaymentProcessor returns success for ₹0 without actually charging anything.
class NullPaymentProcessor {
  process(amount) { return SUCCESS; }
}
Java standard library: Collections.emptyList(), Collections.emptyMap() — these are Null Object pattern. Instead of returning null for an empty collection, return an object that safely supports all list operations (size=0, iteration, etc.).

Null Object vs Optional
Optional = "I'm not sure if value exists — check before using."
Null Object = "Value always exists — just might do nothing."
// Optional — caller must handle absence
Optional<Vehicle> v = factory.get("truck");
v.ifPresent(x -> x.start()); // explicit check

// Null Object — caller just calls
Vehicle v = factory.get("truck");
v.start(); // always safe, no check
When Null Object wins
✔ Null checks scattered across entire codebase
✔ Optional object needs consistent interface
✔ Default/fallback behaviour makes sense

✗ Avoid when null = meaningful absence that must be handled explicitly
Definition: Null Object Pattern replaces null values with an object that implements the same interface but provides default or no-op behaviour.

Key benefit: Eliminates null checks using polymorphism — caller never needs to know if the object is real or null.
