One line: Interpreter evaluates expressions by representing them as a tree of objects — each node interprets its part recursively.

Memory trick: Expression = sentence. Context = dictionary. Interpreter reads both and computes meaning.
Core idea — expression tree
(a * b) + (c * d)
+
↙ ↘
* *
a b c d
Leaves = terminal (values). Internal nodes = non-terminal (operators).
Key insight: Interpreter = Recursion on tree. Every node calls interpret() on its children, combines the result, returns up. Leaf nodes return context values. Non-leaf nodes apply their operation.
4 components — always the same.
1
Abstract Expression — common contract
interface Expression { int interpret(Context ctx); } — every node in the tree implements this.
2
Terminal Expression — leaf nodes
Variables like a, b, 5. No children. Just looks up value in Context. The base case of recursion.
3
Non-Terminal Expression — internal nodes
Operations like +, *, -. HAS-A left + right expression. interpret() calls both children, combines results.
4
Context — the dictionary
Maps variable names to values. a=2, b=4, c=5, d=6. Terminal expressions look up their value here.
// Expression interface interface Expression { int interpret(Context ctx); } // Context — dictionary class Context { Map<String, Integer> map = new HashMap<>(); void put(String k, int v) { map.put(k, v); } int getValue(String k) { return map.get(k); } } // Terminal — leaf node class NumberExpression implements Expression { String key; NumberExpression(String key) { this.key = key; } public int interpret(Context ctx) { return ctx.getValue(key); } } // Non-Terminal — operator nodes class MultiplyExpression implements Expression { Expression left, right; MultiplyExpression(Expression l, Expression r) { left=l; right=r; } public int interpret(Context ctx) { return left.interpret(ctx) * right.interpret(ctx); } } class AddExpression implements Expression { Expression left, right; AddExpression(Expression l, Expression r) { left=l; right=r; } public int interpret(Context ctx) { return left.interpret(ctx) + right.interpret(ctx); } }
// Build tree: (a * b) + (c * d) Context ctx = new Context(); ctx.put("a", 2); ctx.put("b", 4); ctx.put("c", 5); ctx.put("d", 6); Expression expr = new AddExpression( new MultiplyExpression( new NumberExpression("a"), new NumberExpression("b") ), new MultiplyExpression( new NumberExpression("c"), new NumberExpression("d") ) ); System.out.println(expr.interpret(ctx)); // (2*4) + (5*6) = 8 + 30 = 38
Optimised variant — single BinaryExpression class: Trade extensibility for fewer classes. Use a switch on operator string. Good when operators are stable and few.
class BinaryExpression implements Expression { Expression left, right; String operator; // ... public int interpret(Context ctx) { int l = left.interpret(ctx), r = right.interpret(ctx); switch(operator) { case "+": return l + r; case "*": return l * r; case "-": return l - r; } return 0; } }

Interpreter vs Composite: Composite = "all nodes treated same way." Interpreter = "different nodes do different things (terminal vs non-terminal)." Interpreter IS a kind of Composite — but with distinct behaviour per node type.
Definition: Interpreter is a behavioural pattern that evaluates expressions by representing them as a tree of objects, each interpreting its part recursively.
