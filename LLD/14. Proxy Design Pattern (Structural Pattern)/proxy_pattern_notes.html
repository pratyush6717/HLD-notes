4 types of proxy — same structure, different purpose.

Role-based access control. Checks permissions before delegating.

Non-admin ✗ → Exception thrown

Access a remote object as if it's local. Hides network complexity.

// Subject interface
interface EmployeeDao {
  void create(String role, Employee emp);
  void delete(String role, int id);
}

// Real Subject
class EmployeeDaoImpl implements EmployeeDao {
  public void create(String role, Employee emp) {
    // actual DB insert
  }
  public void delete(String role, int id) {
    // actual DB delete
  }
}
// Proxy — Protection proxy with RBAC
class EmployeeDaoProxy implements EmployeeDao {
  private EmployeeDaoImpl realDao = new EmployeeDaoImpl();

  public void create(String role, Employee emp) {
    if (role.equals("ADMIN") || role.equals("MANAGER")) {
      realDao.create(role, emp); // delegate
    } else {
      throw new IllegalAccessException("Not allowed");
    }
  }

  public void delete(String role, int id) {
    if (role.equals("ADMIN")) {
      realDao.delete(role, id); // only admin
    } else {
      throw new IllegalAccessException("Not allowed");
    }
  }
}

// Client — talks only to proxy, never to real object
EmployeeDao dao = new EmployeeDaoProxy();
dao.delete("ADMIN", 101);   // ✔ allowed
dao.delete("USER", 101);    // ✗ exception
IS-A + HAS-A combo: EmployeeDaoProxy IS-A EmployeeDao (implements interface — transparent to client). EmployeeDaoProxy HAS-A EmployeeDaoImpl (holds reference, delegates when allowed).

Spring AOP mechanism: When you annotate a class, Spring doesn't modify your class. It creates a CGLIB or JDK dynamic proxy that wraps your class. That proxy intercepts all method calls. This is the Proxy pattern running in production every time you use Spring.
Definition: Proxy pattern provides a surrogate or placeholder for another object to control access to it.

Key point: Client never interacts with the real object directly.
Proxy vs Decorator vs Adapter — the classic confusion
PatternPurposeInterfaceShortcut question
ProxyControl/restrict access to real objectSame interface"Should I allow this call?"
DecoratorAdd/enhance behaviour dynamicallySame interface"Let me add something on top"
AdapterConvert one interface to anotherDifferent interface"Let me translate this call"
Advantages
  • Access control — security layer
  • Add behaviour without modifying code
  • Lazy loading — performance boost
  • Separation of concerns
Disadvantages
  • More classes — complexity increases
  • Indirection — debugging harder
  • Slight performance overhead
