The pain: Product Service is down. Without circuit breaker, Order Service keeps calling it — every call blocks a thread, waits, fails. Load piles up on the already-down service. Everything cascades.
Without circuit breaker
  • Product down → Order keeps calling
  • Every call waits → threads blocked
  • Adds more load to already-failing service
  • Order Service thread pool fills → crashes too
  • Cascading failure — whole system down
With circuit breaker
  • Failure rate crosses threshold
  • Circuit opens — calls stop immediately
  • No threads blocked waiting
  • Failing service gets breathing room
  • System automatically tests recovery
Aha moment:
Retry = optimistic — keep trying, it might work
Circuit Breaker = realistic — stop trying, it's clearly broken

Together: Retry a few times → if still failing → Circuit Breaker opens → zero calls
Electric circuit analogy: Light switch. CLOSED = switch on, current flows, calls go through. OPEN = switch off, no current, calls blocked immediately. HALF-OPEN = test switch, check if current restored.
3 states define circuit breaker behaviour. Memory trick: Closed circuit = current flows = calls flow. Open circuit = no current = no calls.

Circuit Breaker and Retry solve different problems. Together they form a powerful combo.
Retry — optimistic
Assumes failure is transient. Tries again with backoff. Useful for brief network blips.
✔ Handles occasional errors
✔ Transparent to caller
✗ Makes outage worse if service is truly down
✗ Amplifies load on failing service
Circuit Breaker — realistic
Assumes high failure = real problem. Stops all calls. Gives service time to recover.
✔ Prevents resource waste
✔ Stops cascade failure
✔ Auto-recovers via HALF-OPEN
✗ Wrong threshold → opens too early

Definition: Circuit breaker prevents repeated calls to a failing service by opening the circuit after a failure threshold is reached — reducing resource waste and preventing cascading failures.

Key insight: Three states — Closed (normal), Open (fail fast), Half-Open (testing recovery).

Bonus: Works best combined with Retry — Retry for transient blips, Circuit Breaker for sustained outages.
Full Resilience4j chain — position matters
Rate Limiter → Bulkhead → Time Limiter → Circuit Breaker → Retry
Circuit Breaker before Retry — if circuit is open, Retry never fires. Prevents retry amplification during outage.
