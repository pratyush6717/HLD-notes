🔥 1. Overall idea

👉 JWT auth = login ek baar, baaki requests me token use

🔹 2. Full flow (big picture)
Login:
Client → /auth/login → JWT milta hai

Next requests:
Client → Bearer token bhejta hai
↓
JwtAuthFilter check karta hai
↓
SecurityContext set hota hai
↓
Controller execute ✔
🔥 3. Step-by-step breakdown
🟢 Step 1: Login API
POST /auth/login
username + password

👉 Server:

✔ DB se verify (UserDetailsService)
✔ Password match
✔ JWT generate
🟡 Step 2: JWT return
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

👉 Client store karega ✔

🔵 Step 3: Next request
GET /users
Authorization: Bearer <JWT>
🔥 4. Sabse important part: JwtAuthFilter

👉 Ye har request pe chalta hai

Flow inside filter
1. Header se token nikalo
2. Validate karo (signature + expiry)
3. Username extract karo
4. UserDetails load karo
5. SecurityContext me set karo
🧠 Simple samajh

👉 Filter bolta hai:

"Token valid hai? → haan → user ko login maan lo"
🔹 5. Code ka meaning
🔴 Add filter
http.addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)

👉 Matlab:

JWT check → login filter se pehle ✔
🟣 Stateless
.sessionManagement(s -> s.sessionCreationPolicy(STATELESS))

👉 No session ✔

⚫ CSRF disable
.csrf().disable()

👉 Kyun?

JWT me session nahi hota → CSRF irrelevant ✔
🔥 6. Components samjho (simple)
JwtService
- token banata hai
- token verify karta hai
JwtAuthFilter
- request intercept karta hai
- token check karta hai
AuthController
- login handle karta hai
- token return karta hai
SecurityConfig
- sab connect karta hai

⚡ Final ek line

👉 JWT Auth = filter token check karta hai, session nahi hota

🧠 Interview one-liner

JWT authentication is implemented using a custom filter that extracts and validates the token on each request, 
sets the authentication in the SecurityContext, and allows stateless access to secured endpoints.
