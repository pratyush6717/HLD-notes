🔥 1. OneToMany / ManyToOne (sabse common)
🧠 Example

👉 1 User → multiple Orders

User (1) → Orders (many)
🔹 Database me kaise dikhega?
User Table
------------
id | name

Order Table
------------
id | amount | user_id  👈 FK

👉 FK hamesha “many side” me hota hai

🔥 2. Code samjho
User side (OneToMany)
@OneToMany(mappedBy = "user", cascade = CascadeType.ALL, fetch = FetchType.LAZY)
private List<Order> orders;
Order side (ManyToOne)
@ManyToOne
@JoinColumn(name = "user_id")
private User user;
🔹 3. Owner kaun hai? (VERY IMPORTANT)

👉 Order is OWNER

Kyuki FK order table me hai ✔

👉 Isliye:

@JoinColumn → owner side
mappedBy → non-owner
🔥 4. Easy analogy

👉 Think:

User = Customer
Order = Orders

👉 Order me hi customer_id hota hai ✔

🔹 5. Flow
User save
↓
Orders auto save (cascade)
↓
user_id set in orders table
🔥 6. ManyToMany
🧠 Example

👉 Student ↔ Course

1 student → many courses
1 course → many students
🔹 Database me
Student Table
Course Table
Student_Course (join table) 👈
🔥 7. Code
@ManyToMany
@JoinTable(
    name = "student_course",
    joinColumns = @JoinColumn(name = "student_id"),
    inverseJoinColumns = @JoinColumn(name = "course_id")
)
private List<Course> courses;
🧠 Meaning
Join table create hoga ✔
2 FK honge:
- student_id
- course_id
🔥 8. FetchType (VERY IMPORTANT)
🟢 LAZY (recommended)
fetch = FetchType.LAZY

👉 Data tab load hoga jab use karoge

user.getOrders() → tab query chalegi
🔴 EAGER
fetch = FetchType.EAGER

👉 Automatically load hoga

User aaya → orders bhi aagaye
⚠️ Problem: N+1 issue
1 user → 1 query
10 users → 10 extra queries 😵

👉 Isliye LAZY use karo ✔

🔥 9. Easy summary
OneToMany / ManyToOne
FK = many side me ✔
Owner = many side ✔
ManyToMany
Join table create hota hai ✔
⚡ Final one-line

👉 ManyToOne = real owner (FK wala side)
👉 OneToMany = inverse side (mappedBy)
👉 ManyToMany = join table

🧠 Interview one-liner

In JPA, the ManyToOne side is the owning side containing the foreign key, 
while OneToMany is the inverse side using mappedBy,
 and ManyToMany relationships use a join table to manage associations.



JPA mein relationships ko samajhne ka sabse best tareeka ye hai ki aap Foreign Key (FK) aur Join Table ka logic pakad lein.

Yahan in teeno ka breakdown hai:

1. @ManyToOne (The Most Common)
Ye sabse zyada use hone wala relation hai. Isme "Many" side (bacha) "One" side (baap) ka reference rakhta hai.

Example: Bahut saare Orders ek hi User ke ho sakte hain.

Database: Orders table mein user_id naam ka ek column (Foreign Key) ban jayega.

Owner: Hamesha Many side owner hoti hai kyunki FK wahi hold karti hai.

2. @OneToMany
Ye @ManyToOne ka ulta (bidirectional) hissa hai.

Example: Ek User ke paas bahut saare Orders ki list hai.

Point to Note: Agar aap sirf @OneToMany likhenge bina mappedBy ke, 
toh Hibernate ek faltu ki "Join Table" bana dega. 
Isliye ise hamesha mappedBy ke saath use karein taaki ye sirf ek "view" rahe.

3. @ManyToMany
Jab dono taraf "Many" ka relation ho.

Example: Ek Student bahut saare Courses le sakta hai,
 aur ek Course mein bahut saare Students ho sakte hain.

Database: Isme sirf Foreign Key se kaam nahi chalta.
 Ek Third Table (Join Table) banti hai, jaise student_courses, jisme dono ki IDs hoti hain.
