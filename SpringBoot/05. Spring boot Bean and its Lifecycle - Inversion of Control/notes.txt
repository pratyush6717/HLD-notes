🧠 1️⃣ Bean kya hota hai? (1 line)

👉 Bean = object jo Spring bana ke deta hai

💡 Example

Normal Java:

User user = new User();

Spring:

@Autowired
User user;

👉 Tumne new nahi likha
👉 Spring ne object bana diya = Bean

👉 Object tum nahi banaoge → Spring banayega

🧃 Real Life Example

Maa chai bana ke deti hai → Spring 😄

🧠 3️⃣ @Component kya karta hai?

👉 Spring ko bolta hai:

"Is class ka object bana do"
Example
@Component
class User {}

👉 Bas itna likho → Spring object bana dega

🧠 4️⃣ @Bean kab use karte hai?

👉 Jab tum bolna chahte ho:

"Object kaise banana hai main bataunga"
Example
@Bean
public User createUser() {
    return new User("pk", "email");
}
🔥 Difference (simple)
@Component	@Bean
Spring decide kare	Tum decide karo
simple cases	complex cases

🧠 7️⃣ Lifecycle (sabse simple version)

👉 Bean ka life cycle:

1. Spring start hua
2. Object bana
3. Dependency inject hui
4. Ready to use
5. App band → object destroy
Extra hooks
@PostConstruct  → after creation
@PreDestroy     → before destroy
🧠 8️⃣ Dependency Injection kya hai?

👉 Ek object ko dusra object dena

Example
@Component
class User {
    @Autowired
    Order order;
}

👉 Spring bolega:

User ko Order chahiye → lo ye Order object

Singleton Scope Problem


The Cure: Singleton Scope (The Default)
When Spring sees @Component, it creates exactly one instance of that class during startup and stores it in its 
"Magic Box" (the IoC Container).

The Mechanism: * Developer A asks for the EmailService (via @Autowired). Spring gives them the one in the box.

Developer B asks for the EmailService. Spring gives them the exact same object.

They are both holding a reference to the same memory address.

3️⃣ The "What If" Sandbox: Changing the @Scope
Sometimes, a Singleton is a bad idea. What if you need a fresh object every single time?

The Prototype Scope
If you label your class with @Scope("prototype"), Spring changes its behavior:

The Mechanism: Every time someone asks for that bean, Spring says, 
"Wait, let me make a fresh one for you." It runs the "new" logic specifically for that request.
