🔥 1. Problem: Entity directly return karna ❌

Socho tumhari entity:

@Entity
public class User {
    private Long id;
    private String username;
    private String password; // sensitive ⚠️
}
❌ Controller
@GetMapping("/{id}")
public User getUser() {
    return user;
}
🧠 Problem
- password expose ho gaya ❌
- extra fields bhi aa sakte hain ❌
- DB structure leak ho raha ❌
🔥 2. Solution = DTO

👉 DTO = Data Transfer Object (sirf required data)

✅ DTO class
public class UserDTO {
    private Long id;
    private String username;
}
✅ Controller
@GetMapping("/{id}")
public UserDTO getUser() {
    User user = service.getUser(id);
    return convertToDTO(user);
}
🧠 Result
Safe ✔
Clean ✔
Only required data ✔
🔹 3. Mapping kaise kare?
🟢 Manual mapping
UserDTO dto = new UserDTO();
dto.setId(user.getId());
dto.setUsername(user.getUsername());

👉 Simple but repetitive ❌

🟡 ModelMapper (easy)

👉 ModelMapper

ModelMapper mapper = new ModelMapper();
UserDTO dto = mapper.map(user, UserDTO.class);
🔴 MapStruct (fast, compile-time)

👉 MapStruct

👉 Production me zyada use hota hai ✔

🔥 6. Real flow (important)
DB → Entity → DTO → API response
🔹 7. Real-life analogy

👉 Entity = full database record 📂
👉 DTO = filtered data 📄

Client ko sirf zaroori info do ✔
⚡ Final ek line

👉 DTO = API ke liye clean & safe data

🧠 Interview one-liner

DTOs are used to transfer only required data to the client, avoiding exposure of entity structure and sensitive fields, 
and are typically mapped using tools like ModelMapper or MapStruct.
